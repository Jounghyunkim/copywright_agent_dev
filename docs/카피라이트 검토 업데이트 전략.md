# 카피라이트 검토 업데이트 전략

각국 법인 마케터가 직접 작성한 카피라이트를 다양한 업무 Skill로 검수·수정·업데이트하는 "카피라이트 검토" 기능의 고도화 전략 문서.

---

## 1. 현황 진단

### 강점
- 2단계(입력 → 검토) 골격과 스킬 기반 re-review 루프가 동작
- 20개 국가 플래그, 카피 필드(Headline / Subheadline / Body / CTA) 구조화 입력
- SSE 기반 실시간 리뷰 진행 상황, 보정 후 재평가까지 구현

### 주요 결핍
1. **컨텍스트 공백** — `REVIEW_ONLY_ANALYSIS`·`REVIEW_ONLY_STRATEGY`가 빈 객체로 전달되어 brand-fit / tone / segment-fit 같은 컨텍스트 의존 스킬이 제 성능을 못 냄
2. **스킬 큐레이션 부재** — 56개 스킬이 평면 리스트. 법인 마케터가 "내 상황에 뭘 돌려야 하나"를 판단할 수 없음
3. **국가-스킬 자동 매핑 없음** — 국가 선택해도 `culture-<country>`, 규제성 스킬이 자동 추천되지 않음
4. **목적(Intent) 미수집** — 브랜딩 / 프로모션 / 성능 / 법무 감수 등 목적 구분이 없어 스킬 선택 편향이 생김
5. **원본·레퍼런스 부재** — 번역 이슈나 현지화 품질을 판정할 기준(원 카피, 글로벌 마스터 카피)을 입력할 곳 없음
6. **리포트 산출 부실** — 법무/브랜드 위원회로 넘길 PDF/CSV export, 심각도 라벨링(critical/warn/info)이 없음
7. **협업·감사 미지원** — 승인 라우팅, 코멘트 스레드, 공유 링크, 수정 히스토리 없음
8. **배치/대량 입력 취약** — 한 건씩 폼 입력뿐. 법인이 XLSX/CSV로 벌크 업로드·검증하는 실사용 경로가 없음

---

## 2. 핵심 문제 정의

> "법인 마케터가 자신의 카피를 **가장 빠르게**, **가장 근거 있게**, **가장 안전하게** 검증·개선할 수 있는가?"

3축(Speed / Evidence / Safety) 관점에서 현재 수준:

| 축 | 현재 | 목표 |
|----|------|------|
| Speed | 中 | 上 (유지 + 벌크 가속) |
| Evidence | 下 | 上 (근거·출처 제공) |
| Safety | 下 | 上 (심각도·게시 차단) |

고도화는 Evidence와 Safety를 끌어올리되 Speed를 희생하지 않는 방향.

## 3. 4개 전략 축

### 축 A. 리뷰 레인(Lane) 구조화 — "무엇을 검증하는가"

플랫 스킬 리스트 → **4개 레인**으로 재정렬

| 레인 | 목적 | 핵심 스킬 | 산출물 |
|------|------|-----------|--------|
| ① Risk & Compliance | 법무·규제 리스크 감지 | `compliance-redflag-detector`, `ai-washing-risk-check`, `environmental-claim-risk-check`, `comparative-ad-risk-check`, `claim-extractor`, `proof-point-checker`, `regulatory-copy-validation` | 위반 조항·근거 문구·심각도 |
| ② Brand Integrity | LG 브랜드 정합성 | `lg-brand-fit-check`, `lg-brand-voice`, `brand-lexicon-check`, `tone-and-voice-enforcer` | Brand Fit Score + 수정 포인트 |
| ③ Craft Quality | 카피 자체 품질 | `main-message-clarifier`, `headline-body-cta-composer`, `creative-impact-scorer`, `copy-scorecard-generator`, `customer-segment-fit-check` | 10점 척도 + 개선안 |
| ④ Localization | 국가·문화 적합성 | `language-transcreation`, `regional-copy-adaptation`, `regional-culture-fit-check`, `culture-<country>` | 현지화 교정 + 문화 리스크 |

각 레인은 **Preset**(Light / Standard / Deep)로 묶어 클릭 한 번에 관련 스킬 실행.

### 축 B. 컨텍스트 보강 — "무엇을 근거로 판정하는가"
- **입력 스키마 확장**: `campaignIntent`(branding|promotion|performance|launch), `productCategory`, `originalCopy`(원문/마스터 카피), `targetAudience`, `regulatoryContext`(예: "FTC, EU DSA 적용")
- **브랜드/페르소나 자동 주입**: 저장된 최근 캠페인의 `analysisReport`/`strategicMessage`를 "참조 캠페인"으로 연결 → 리뷰 API에 실제 컨텍스트 전달
- **Lexicon 파일 업로드**: 금지어·선호어 사전(.csv)을 업로드해 `brand-lexicon-check`에 주입
- **국가 자동 프로파일링**: 국가 선택 시 `culture-<country>` + 해당 국가 규제 번들이 자동 추천

### 축 C. 입력·UX 고도화 — "어떻게 빨리 올리는가"
- **벌크 업로드**: XLSX/CSV 템플릿 제공(1행=1카피, 컬럼: country / lang / headline / ... / intent) → 파싱 프리뷰 → 일괄 리뷰
- **Figma/Slack 링크 파싱**(2차): URL 붙여넣기로 스크린샷 + 텍스트 자동 추출
- **다국어 카피 언어 감지**: 입력 언어 자동 판별 → 해당 언어 전용 스킬 세트 적용
- **비교 뷰**: 원문 ↔ 번역본 ↔ AI 제안 3-column diff
- **A/B 변형 관리**: 같은 의도의 복수 변형을 한 단위로 묶어 스코어 나란히 비교

### 축 D. 리뷰 결과 고도화 — "어떻게 의사결정에 연결하는가"
- **심각도 분류**: 모든 Finding에 `critical` / `warning` / `suggestion` 부여 (critical이 1개라도 있으면 게시 불가 플래그)
- **Evidence Trail**: 각 지적 사항에 근거(법조항 / 브랜드 가이드 섹션 / 유사 사례)를 스킬이 출처와 함께 반환
- **Rewrite Plan**: 여러 스킬의 지적을 통합해 "이렇게 고치면 모든 문제 해결" 식의 단일 수정안 생성 — 메타 스킬 `copy-rewrite-synthesizer` 신설
- **리포트 Export**: 법무/브랜드 위원회용 PDF(국가별 페이지) + 실무용 CSV
- **승인 라우팅**: 카피별 상태(draft / under-review / approved / rejected) + 담당자 지정. 관리자 페이지의 사용자 권한과 연결
- **이력·감사 로그**: 리뷰·수정·승인 시각 + 사용자 기록 (백엔드 `auth_session_events` 테이블 골격 존재 → 확장)

## 4. 로드맵 (3-Phase)

### Phase 1 — 기초 다지기 (2주)
현재 코드 구조 훼손 최소화하며 가치 크게 상승
- **[1-A]** 입력 스키마 확장: `campaignIntent`, `originalCopy`, `productCategory` 필드 추가
- **[1-B]** 레인 프리셋 UI: 스킬 리스트를 4레인 탭으로 재배치 + "Standard Preset" 버튼
- **[1-C]** 국가 자동 스킬 추천: 국가 선택 시 `culture-<country>` 자동 체크
- **[1-D]** 심각도 라벨: 백엔드 리뷰 응답에 `severity` 필드 추가, ReviewResults에 컬러 뱃지

### Phase 2 — 컨텍스트·검증 강화 (3주)
- **[2-A]** "참조 캠페인" 연결: 기존 campaigns에서 하나 선택 → analysisReport / strategicMessage 자동 주입
- **[2-B]** Lexicon 업로드(브랜드 사전)
- **[2-C]** 벌크 업로드(XLSX)
- **[2-D]** `copy-rewrite-synthesizer` 메타 스킬 신설: 여러 Finding 통합 → 단일 Rewrite Plan
- **[2-E]** 언어 자동 감지 + 다국어 스킬 경로 분리

### Phase 3 — 협업·리포팅 (3주)
- **[3-A]** 심각도 기반 Publish Gate (critical 발견 시 승인 차단)
- **[3-B]** PDF 리포트 Export (국가별 섹션, 심각도 요약, 원본/수정본 diff)
- **[3-C]** 승인 라우팅 + 코멘트 스레드
- **[3-D]** 감사 로그 UI (관리자 페이지에 탭 추가)
- **[3-E]** Feedback loop: 사용자가 수용/거절한 Finding을 집계 → 각 스킬의 "수용률" 대시보드

## 5. 우선순위 제안

### 지금 바로 착수 권장 (ROI 高, 공수 小)
1. **입력 스키마에 `campaignIntent` + `originalCopy` 추가** — 전체 리뷰 품질 향상의 선제조건
2. **레인 프리셋 UI** — 사용자가 "무엇을 돌려야 하나" 고민 없이 한 클릭으로 의미 있는 조합 선택
3. **국가 자동 스킬 추천 + 심각도 라벨** — 즉각적 UX / 의사결정 가치

### 중기 (2~4주 내)
4. **`copy-rewrite-synthesizer` 메타 스킬** — 개별 지적에서 통합 수정안으로 도약. 이것이 도구를 "감수기"에서 **"코치"**로 격상시키는 핵심.
5. **참조 캠페인 연결**로 컨텍스트 공백 해결

### 장기 (1~2개월)
6. 벌크 업로드, PDF Export, 승인 라우팅

## 6. 지표(Success Metrics)
- **카피 1건 리뷰 완료 시간**: 현행 대비 50% 단축 (레인 프리셋 효과)
- **Critical 발견률**: 출시 전 치명 리스크 감지율
- **Rewrite 수용률**: AI 수정안이 최종 게시 버전에 반영된 비율 (스킬 품질 지표)
- **법인별 사용 빈도**: 사용 통계 대시보드에서 국가/부서별 DAU 추적

## 7. 포지셔닝 전환

> "스킬 실행기" → **"브랜드 감수관 (AI Brand Guardian)"**

특히 다음 3가지가 전환점이 될 축:
1. **레인 구조화** (Lane Structure)
2. **심각도 라벨** (Severity Tagging)
3. **Rewrite Synthesizer** (통합 수정 제안)
