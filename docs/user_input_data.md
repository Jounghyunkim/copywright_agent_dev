# Copywriting Agent 사용자 입력 항목 정의서

## 개요

Copywriting Agent 플랫폼에서 사용자로부터 입력받는 전체 항목을 정리한 문서입니다.
총 **41개 항목** (필수 29개 / 선택 12개)이며, 7개 영역으로 분류됩니다.

---

## 1. 카피라이트 생성 — 브리핑 단계

> 경로: `/workflow` > 좌측 패널 > BriefingForm
> 9개 섹션으로 구성된 접이식 폼. 각 섹션에 (?) 가이드 버튼이 있어 AI 어시스턴트가 작성 도움말을 제공합니다.

| # | 필드명 | 타입 | 필수 | 설명 |
|---|--------|------|:---:|------|
| 1 | **Project Name** | text | ✅ | 캠페인 프로젝트명 (예: LG OLED TV Global Launch) |
| 2 | **Project Context** | textarea | ✅ | 프로젝트 배경 및 중요성. 신제품 런칭 / 소비자 인사이트 / 경쟁사 대응 / 시즌 이벤트 / 브랜드 전략 등 |
| 3 | **Commercial Objective** | textarea | ✅ | 비즈니스 목표 — 매출 증대, 시장 점유율, 재구매율 등. 가능하면 수치 목표 포함 |
| 4 | **Behavior Objective** | textarea | ✅ | 행동 목표 — 광고를 본 소비자가 취하길 바라는 행동 (웹사이트 방문, 매장 예약, 콘텐츠 공유 등) |
| 5 | **Attitudinal Objective** | textarea | ✅ | 인식 목표 — 브랜드/제품에 대한 소비자 인식 변화 (인지도, 선호도, NPS 등) |
| 6 | **Audience** | textarea | ✅ | 핵심 타겟 정의 — 인구통계(연령/성별/소득), 라이프스타일, 구매 행동, 페인 포인트. Primary/Secondary 구분 권장 |
| 7 | **Key Message** | textarea | ✅ | 소비자에게 전달할 단 하나의 핵심 메시지 (1-2문장). 기능(Feature)이 아닌 혜택(Benefit) 관점으로 작성 |
| 8 | **Proof Points** | textarea | ✅ | Key Message를 뒷받침하는 근거 1~3개 (기술적 우위, 수상/인증, 실적, 소비자 증언, 전문가 보증) |
| 9 | **Mandatories** | textarea | - | 필수 준수 요건 — 미디어/채널, 제작물 스펙, 모델/앰배서더, 브랜드 가이드라인, 법적/규제 사항 |
| 10 | **Budget** | text | - | 캠페인 전체 예산 또는 매체별 예산 배분. 전년 대비 증감 현황 포함 권장 |
| 11 | **Market Needs** | textarea | ✅ | 타겟 국가/지역, 언어 Variation, 문화적 고려사항, 현지 경쟁 환경, 미디어 환경 차이 |
| 12 | **Timing** | text | ✅ | 캠페인 집행 기간, 주요 마일스톤(제품 출시일, CES/IFA 등), 단계별 계획(Teasing/Launch/Sustain) |
| 13 | **Message Matrix** | file (.xlsx/.xls) | - | 제품 USP 정보가 담긴 엑셀 파일. 업로드 시 시트 선택 → 파싱 → Project Name과 Project Context를 자동으로 채움 |

### 참고: AI 자동생성

- **Project Name**과 **Project Context**를 입력한 뒤 "AI 자동생성" 버튼을 누르면 나머지 9개 필드를 LLM이 자동으로 채워줍니다.
- 자동생성 조건: Project Name 3자 이상, Project Context 20자/5단어 이상.

## 2. 카피라이트 생성 — 카피 생성 조건 (Step 4)

> 전략 메시지 승인 후 표시되는 GenerationConfig 폼.

| # | 필드명 | 타입 | 필수 | 설명 |
|---|--------|------|:---:|------|
| 14 | **타겟 국가** | 다중선택 체크박스 (20개국) | ✅ | US, DE, GB, FR, IT, ES, JP, CN, IN, BR, KR, AU, ID, TH, SA, NL, SE, PL, MX, CA. "전체 선택/해제" 토글 지원 |
| 15 | **타겟 연령대** | 다중선택 체크박스 (5그룹) | ✅ | 18-24 / 25-34 / 35-44 / 45-54 / 55+ |
| 16 | **타겟 페르소나** | 다중선택 카드 (5종) | ✅ | Tech Enthusiast (얼리어답터) / Premium Lifestyle (프리미엄 지향) / Value Seeker (가성비 중시) / Family First (가족 중심) / Eco Conscious (환경 관심) |
| 17 | **카피 변형 수** | number input (1-10) | ✅ | 국가당 생성할 카피 변형 개수. 기본값 3 |

---

## 3. 카피라이트 생성 — 리뷰 단계 (Step 5)

> 생성된 카피를 스킬 기반으로 평가하고, 개선안을 선택하여 카피를 보정하는 HITL 단계.

| # | 필드명 | 타입 | 필수 | 설명 |
|---|--------|------|:---:|------|
| 18 | **카피 선택** | 체크박스 (다중) | ✅ | Copy Results에서 리뷰 대상으로 선택할 카피. 국가-변형별 체크 (예: US-0, US-1, KR-0) |
| 19 | **리뷰 스킬 선택** | 체크박스 (다중) | ✅ | 카피에 적용할 검증/분석 스킬 선택. built-in(SKILL.md 기반) + custom(사용자 생성) 모두 표시. "전체 선택/해제" 토글 지원 |
| 20 | **개선안 선택 (IMPROVEMENTS)** | 체크박스 (다중) | - | 리뷰 결과에서 도출된 개선 제안 항목 중 카피 보정에 적용할 항목을 선택. 선택 후 "선택한 개선안으로 카피 보정" 클릭 시 LLM이 카피를 수정하고 재평가 자동 실행 |

## 4. 카피라이트 검토 — 카피 직접 입력

> 경로: `/copy-review` > 좌측 패널
> 기존에 작성된 카피를 직접 입력하여 스킬 기반 검토만 수행하는 독립 워크플로우.
> "카피 추가" 버튼으로 여러 국가/변형의 카피를 동시에 입력 가능.

| # | 필드명 | 타입 | 필수 | 설명 |
|---|--------|------|:---:|------|
| 21 | **국가** | select 드롭다운 (20개국) | ✅ | 카피가 사용될 타겟 국가. 국기 + 국가명 + 언어 표시 |
| 22 | **Headline** | text | ✅ | 메인 헤드라인 |
| 23 | **Subheadline** | text | ✅ | 보조 헤드라인 |
| 24 | **Body Copy** | textarea | ✅ | 본문 카피 |
| 25 | **CTA** | text | ✅ | Call to Action 문구 (예: "자세히 보기", "지금 구매하기") |
| 26 | **Methodology** | text | - | 카피 작성에 사용된 방법론 (접이식 선택 항목) |
| 27 | **Cultural Notes** | text | - | 해당 국가/문화권에 대한 고려사항 (접이식 선택 항목) |
| 28 | **Tone Analysis** | text | - | 카피의 톤 분석 정보 (접이식 선택 항목) |

### 참고

- 입력 완료 후 "입력 완료" 클릭 → Copy Results 카드로 표시 → 리뷰 조건 설정 → 스킬 리뷰 실행.
- 리뷰 플로우(스킬 선택, 개선안 체크, 카피 보정)는 "카피라이트 생성"과 동일.
- 비한국어 카피는 자동으로 한국어 번역이 함께 표시됨.

## 5. 스킬 관리 — 새 스킬 생성 / 편집

> 경로: `/skills` > "새 스킬" 버튼 또는 기존 스킬 "편집" 클릭 시 모달 표시.
> built-in 스킬(SKILL.md 기반)은 읽기 전용, custom 스킬만 편집 가능.

| # | 필드명 | 타입 | 필수 | 설명 |
|---|--------|------|:---:|------|
| 29 | **ID** | text (kebab-case 자동변환) | ✅ | 스킬 고유 식별자 (예: `promo-legal-check`). 3자 이상, 저장 후 변경 불가 |
| 30 | **라벨** | text | ✅ | UI에 표시되는 스킬 이름 (예: 프로모션 법률 검증) |
| 31 | **설명** | textarea | ✅ | 스킬이 수행하는 작업에 대한 간결한 설명 |
| 32 | **카테고리** | select | ✅ | `validation` (검증) / `generation` (생성) / `analysis` (분석) 중 선택 |
| 33 | **Prompt Template** | textarea (JetBrains Mono) | ✅ | LLM에 전달될 프롬프트 템플릿. `{{copy}}`, `{{brief}}` 등 변수 사용 가능 |
| 34 | **Output Schema** | textarea (JSON, JetBrains Mono) | - | 스킬 결과물의 JSON 스키마 정의. 유효한 JSON 객체여야 함 |
| 35 | **활성화** | checkbox | - | 스킬 활성/비활성 토글. 편집 모드에서만 표시 |

## 6. 스킬 관리 — AI 초안 생성 위자드

> 경로: `/skills` > "AI 초안 생성" 버튼 클릭 시 모달 표시.
> 간단한 정보 입력 → AI가 ID/라벨/설명/Prompt Template/Output Schema 초안을 자동 생성 → 편집 모달로 전달.

| # | 필드명 | 타입 | 필수 | 설명 |
|---|--------|------|:---:|------|
| 36 | **스킬 이름** | text | ✅ | AI가 ID와 라벨을 도출할 기반 이름 (2자 이상) |
| 37 | **작성 목적** | textarea | ✅ | 왜 이 스킬을 만드는지 — 배경과 동기 (10자 이상) |
| 38 | **스킬 목적** | textarea | ✅ | 스킬이 달성해야 하는 구체적 목표 (10자 이상) |
| 39 | **좋은 예시** | textarea | - | 이 스킬이 "통과"시켜야 할 좋은 카피 예시 |
| 40 | **나쁜 예시** | textarea | - | 이 스킬이 "잡아야" 할 나쁜 카피 예시 |

## 워크플로우별 입력 흐름

### 카피라이트 생성 (End-to-End)

```
① 브리핑 입력 (13항목)
   ↓ [② 분석 시작]
   자동: Market Analyst 분석 (SSE)
   ↓ [③ 전략 메시지 생성]
   자동: Copywriting Strategy 추출
   ↓ [④ 카피 후보 생성]
   카피 생성 조건 입력 (4항목)
   ↓ 자동: 카피 생성
   카피 선택 (체크박스)
   ↓ [⑤ 선택 카피 검토]
   리뷰 스킬 선택 (체크박스)
   ↓ [⑥ 리뷰 시작]
   자동: 스킬 기반 리뷰 (SSE)
   ↓ 개선안 선택 → 카피 보정 → 재평가 (반복 가능)
   ↓ [저장 및 종료]
```

### 카피라이트 검토 (카피 직접 입력)

```
① 카피 직접 입력 (8항목 × N개)
   ↓ [입력 완료]
   리뷰 스킬 선택 (체크박스)
   ↓ [② 리뷰 시작]
   자동: 스킬 기반 리뷰 (SSE)
   ↓ 개선안 선택 → 카피 보정 → 재평가 (반복 가능)
   ↓ [저장 및 종료]
```
