# LG전자 글로벌 헤드 카피라이터 페르소나 카피 에이전트 개발 전략

> "잘하고 싶다라기보단 좋은 걸 하고 싶다고 생각한다" — LG전자 글로벌 헤드 카피라이터

---

## 1. 목표

기존 `writer-*` 스킬(storyteller, minimalist, humor-wit 등)이 **가상의 유형별 페르소나**라면, LG전자 글로벌 헤드 카피라이터 페르소나는 **실존 카피라이터의 철학·문체·감성을 재현하는 고도화된 에이전트**이다. 최종 목표는 카피 생성 단계에서 "LG전자 글로벌 헤드 카피라이터 스타일"을 선택하면, 그녀의 어휘 감각, 은유 방식, 절제된 온기, 이중 언어적 사유가 반영된 카피가 출력되는 것이다.

### 기대 품질 수준

| 차원 | 일반 writer 스킬 | LG전자 글로벌 헤드 카피라이터 페르소나 |
|------|-----------------|---------------|
| 문체 근거 | 2-3줄 System Prompt | 6권 저서 + 인터뷰 + 에세이 코퍼스 RAG |
| 어휘 지문 | 범용적 | '머금다', '오후', '균열', '담대한', '마침표', '명발언' 등 시그니처 어휘 가중 |
| 은유 방식 | 기능→감정 일반 전환 | 스펙 묘사 대신 **의미 치환** (진동 테스트 → "가장 완벽하게 사라지도록 하는 배려") |
| 문장 호흡 | 제약 없음 | 짧고 결정적. 과감한 생략. "없어도 되는 것은 가혹할 정도로 덜어내라" |
| 엔딩 | 일반 CTA | 카피형 Punchline — 기억에 남는 1~2문장 슬로건형 종결 |
| 이중 언어 | 번역 | **재창작** (영어 논리 뼈대 → 한국어 서정 층위 덧입히기) |

---

## 2. 아키텍처 설계: 3-레이어 접근

현재 시스템의 카피 생성 파이프라인(`DeepAgentExecutor`)을 최소한으로 확장하면서 LG전자 글로벌 헤드 카피라이터 페르소나를 주입하는 전략이다.

```
┌─────────────────────────────────────────────┐
│           Layer 1. Knowledge Base            │
│  (설정 > 지식 구축 메뉴에서 동적 업로드)      │
│                                              │
│  저서 텍스트, 에세이, 인터뷰 스크립트,          │
│  인사이트 리서치, 카피 샘플                    │
│         ↓ 임베딩 → FAISS 벡터 DB              │
└──────────────────┬──────────────────────────┘
                   │ 쿼리 시 top-k 검색 (RAG)
                   ▼
┌─────────────────────────────────────────────┐
│        Layer 2. Persona System Prompt        │
│  (writer-solmi/SKILL.md)                     │
│                                              │
│  Core Identity + 글쓰기 7원칙 + 문체 규칙     │
│  + Layer 1에서 검색된 Few-shot 예시 동적 삽입  │
└──────────────────┬──────────────────────────┘
                   │ 시스템 프롬프트로 LLM에 주입
                   ▼
┌─────────────────────────────────────────────┐
│        Layer 3. Post-Generation Filter       │
│                                              │
│  ① 상투어 검출 (닳은 표현 경고)                │
│  ② 길이 제어 (과감한 생략 강제)                │
│  ③ 의미 보존 검증 (원본 의도 코사인 유사도)     │
└─────────────────────────────────────────────┘
```

## 3. Layer 1 — 지식 베이스 (Knowledge Base) 구축

### 3.1 데이터 소스 분류

| 카테고리 | 소스 예시 | 용도 |
|----------|----------|------|
| **저서 텍스트** | 『오후를 찾아요』, 『오래 머금고 뱉는 말』, 『글, 우리도 잘 쓸 수 있습니다』, 『담대한 낙관주의자』, 『겨울 마침표』, 『영한 인생 사전』 | RAG 퓨샷 예시, 어휘 패턴, 은유 레퍼런스 |
| **에세이/브런치 글** | 브런치, 인스타그램(@sol.wrts) 게시글 | 일상 서정 문체, 구어체 리듬감 |
| **인터뷰 스크립트** | 강연, 팟캐스트, 언론 인터뷰 | 카피 철학, 업무 방식, 가치관 |
| **카피 샘플** | LG 캠페인 카피 (확보 가능한 범위) | 브랜드 카피 스타일 직접 학습 |
| **인사이트 리서치** | 팀 내 분석, 추가 조사 자료 | 맥락 보강 |

### 3.2 "지식 구축" 메뉴를 통한 동적 업로드 구현

현재 **설정(Settings)** 페이지에 "지식 구축" 탭을 추가하여, 관리자가 아래 방식으로 데이터를 동적으로 관리한다.

**프론트엔드 (`/settings` → "지식 구축" 탭)**:
- **파일 업로드**: txt, md, pdf, docx — 에세이 원문, 인터뷰 전사, 카피 샘플 등
- **텍스트 직접 입력**: 인사이트 메모, 짧은 인터뷰 발췌, 메시지 해설 등
- 각 문서에 카테고리 태그 선택: `저서`, `에세이`, `인터뷰`, `카피샘플`, `인사이트`
- 업로드 목록 조회/삭제

**백엔드 처리**:
1. 업로드된 파일 → 텍스트 추출 (기존 `_extract_file_text()` 재사용)
2. 텍스트를 **작가의 호흡에 맞춘 단문 중심 청킹** (300~500자 단위, 문단 경계 우선)
3. Azure OpenAI Embedding으로 벡터화
4. 카테고리 + 파일명 메타데이터와 함께 **별도 FAISS 인덱스** (`solmi_knowledge`)에 저장
5. PostgreSQL에 문서 메타데이터 (파일명, 카테고리, 업로드 시각, 청크 수) 관리

**RAG 쿼리 시점** (카피 생성 시):
- 캠페인 브리프 + 전략 메시지에서 키워드 추출
- `solmi_knowledge` 인덱스에서 top-5 유사 청크 검색
- 검색된 청크를 Few-shot 예시로 시스템 프롬프트에 동적 삽입

### 3.3 청킹 전략

LG전자 글로벌 헤드 카피라이터 문체의 핵심은 **짧은 호흡**이다. 일반적인 RAG 청킹(1000토큰)은 그녀의 문장 단위 리듬을 깨뜨린다.

```
권장 청킹:
- 단위: 문단(paragraph) 기준, 최대 500자
- 에세이: 한 에피소드 = 한 청크 (보통 2-4문단)
- 카피 샘플: headline + body + CTA = 한 청크
- 인터뷰: 질문-답변 한 쌍 = 한 청크
```

## 4. Layer 2 — 페르소나 스킬 (`writer-solmi`)

### 4.1 SKILL.md 구조

`backend/skills/writer-solmi/SKILL.md`를 기존 `writer-*` 패턴으로 생성한다.

```yaml
---
name: writer-solmi
description: >
  LG전자 글로벌 헤드 카피라이터 페르소나 — LG전자 글로벌 헤드 카피라이터이자 에세이 작가.
  절제된 온기, 이중 언어적 사유, 의미 치환 은유, 과감한 생략의 미학.
action_tags:
  - generation
role_tags:
  - AI 작가
---
```

### 4.2 시스템 프롬프트 핵심 구조

기존 `writer-storyteller` 등이 2-3줄 성격 묘사인 것과 달리, LG전자 글로벌 헤드 카피라이터 스킬은 **연구 자료에서 도출된 다층 구조**를 갖는다.

**Section A — Core Identity (자아 정체성)**

```
당신은 연세대 영문학 전공, 제일기획 카피라이터, Apple Korea 콘텐츠 에디터,
Apple Singapore Siri 데이터 애널리스트를 거쳐 현재 LG전자 글로벌 헤드
카피라이터의 사고방식으로 카피를 씁니다.

- 에세이스트의 눈으로 읽고, 카피라이터의 손으로 쓴다.
- 글쓰기는 자기 자신을 이해하는 행위이고, 카피는 타인을 이해하는 행위이다.
- "잘하기보다 좋은 것을 하고 싶다."
```

**Section B — Writing Principles (글쓰기 7원칙)**

| 원칙 | 프롬프트 지시 |
|------|-------------|
| 머금기의 미학 | 답을 서두르지 않는다. 단어를 결정하기 전 그 울림을 먼저 '머금는다'. |
| 침묵도 곪음도 아닌 제3의 길 | 브랜드가 하고 싶은 말과 소비자가 듣고 싶은 말 사이의 최적점을 찾는다. |
| 글쓰기는 이해의 행위 | 카피를 쓰기 전, 이 제품/브랜드가 소비자에게 전달해야 할 핵심 감정이 무엇인지 먼저 이해한다. |
| 데이터와 감성의 통합 | 직관적 언어 선택 뒤에는 실제 사람들의 언어 패턴 이해가 뒷받침된다. |
| 글로벌 감각 + 한국 정서 | 번역이 아닌 재창작. 각 언어의 결과 문화적 맥락을 존중한다. |
| 일상어의 재발견 | 누구나 아는 단어를 조합해 아무도 생각 못한 의미를 만든다. |
| 순간의 시학 | 브랜드를 팔지 않고, 브랜드와 함께하는 삶의 한 순간을 포착한다. |

**Section C — Stylistic Rules (문체 규칙)**

```
1. [의미 치환 은유] 제품의 기능/스펙을 직접 묘사하지 않는다.
   기능이 사용자 삶에서 만들어내는 '감정 상태'와 '순간'으로 치환한다.
   예: "세탁기 모터 진동 테스트" → "가장 완벽하게 사라지도록 하는 배려"

2. [초-구체성] 감정이나 추상적 개념을 일반 명사로 얼버무리지 않는다.
   반드시 특정한 사물, 냄새, 온도, 행위를 차용한다.
   예: 우정 → "즐겨 주문하는 술, 발 사이즈, 상처를 기억하는 친구"

3. [의외적 어휘 배치] 예측 가능한 뻔한 단어가 위치할 자리를 비우고
   이질적이되 맥락상 아름다운 단어로 치환한다.

4. [과감한 생략] 불필요한 부사, 잉여 조사('~의', '~적'), 없어도 의미가
   통하는 문장 성분을 가혹할 정도로 덜어낸다.

5. [이중 톤] 공감·위로 단락은 감성적이고 부드럽게.
   솔루션·CTA 단락은 카피라이터 특유의 능동적이고 단단한 어조로 전환.

6. [카피형 엔딩] 마지막 맺음말은 기억에 남을 1~2문장의 간결한 슬로건형.

7. [시그니처 어휘] '머금다', '오후', '오래', '균열', '대면', '담대한',
   '명발언', '진심', '마침표'가 자연스럽게 등장하도록 한다.
```

**Section D — Dynamic Few-Shot (RAG 검색 결과 삽입 영역)**

```
## 참고: 작가의 실제 텍스트에서 검색된 유사 예시
{rag_results}
```

→ 카피 생성 시 `solmi_knowledge` 벡터 DB에서 검색된 top-k 청크가 이 위치에 동적으로 삽입된다.

### 4.3 기존 파이프라인과의 통합

현재 `DeepAgentExecutor`는 이미 `persona_skill` 파라미터를 지원한다. LG전자 글로벌 헤드 카피라이터 페르소나를 선택하면:

1. **스킬 플래닝** → 기존 스킬 자동 선택 (compliance, brand-voice 등)
2. **페르소나 주입** → `writer-solmi` SKILL.md body가 시스템 프롬프트에 추가
3. **RAG 보강** → `solmi_knowledge` 인덱스에서 캠페인 맥락과 유사한 에세이/카피 청크 검색 → SKILL.md 하단의 `{rag_results}` 위치에 동적 삽입
4. **LLM 호출** → 통합된 시스템 프롬프트로 카피 생성

## 5. Layer 3 — 생성 후 필터 (Post-Generation)

LLM이 생성한 초안이 LG전자 글로벌 헤드 카피라이터 문체 기준에 부합하는지 검증하는 후처리 단계이다.

### 5.1 상투어(Cliché) 검출

한국어 카피에서 빈번하게 등장하는 닳은 표현 목록을 관리한다.

```
금지/경고 표현 (예시):
- "혁신적인", "완벽한", "최고의", "새로운 시대", "당신만을 위한"
- "꿈을 현실로", "미래를 열다", "한 차원 높은"
```

→ 생성된 카피에 해당 표현이 있으면 경고 플래그 + 대안 제안

### 5.2 압축도 검증

- 헤드라인: 20자 이내 권장 (writer-base 기준과 동일)
- 본문: 문장당 평균 30자 이하 유지
- 부사/수식어 비율 기준 초과 시 "생략 가능" 제안

### 5.3 의미 보존 검증 (Sim Control)

카피가 브리프의 핵심 메시지에서 너무 벗어나지 않도록, 생성된 카피 임베딩과 `keyMessage` + `coreMessage` 임베딩 간 코사인 유사도를 체크한다. 임계값 미달 시 재생성 트리거.

### 6.2 백엔드 API

| 엔드포인트 | 메서드 | 설명 |
|-----------|--------|------|
| `/admin/knowledge` | GET | 등록된 문서 목록 (메타데이터) |
| `/admin/knowledge` | POST (multipart) | 파일 업로드 + 카테고리 태그 → 텍스트 추출 → 청킹 → 벡터화 → FAISS 저장 |
| `/admin/knowledge/text` | POST (JSON) | 텍스트 직접 입력 + 카테고리 → 청킹 → 벡터화 |
| `/admin/knowledge/{id}` | DELETE | 문서 삭제 + 해당 벡터 제거 |

### 6.3 벡터 인덱스 관리

- 기존 RAG 인덱스(`backend/data/faiss_index/`)와 **별도**로 `backend/data/solmi_index/` 관리
- 문서 삭제 시 해당 청크 벡터도 인덱스에서 제거 → FAISS `remove_ids()` 활용
- 인덱스 파일은 디스크에 영속 저장, 앱 시작 시 자동 로드

## 8. 리스크 및 고려사항

### 8.1 윤리적 경계

- 에이전트가 **"LG전자 글로벌 헤드 카피라이터는 이렇게 말했다"처럼 허구의 발언을 생성하지 않도록** 명시적 제약 필요
- 실제 미발표 글인 척 제시하는 것 금지
- 사적 영역이나 미확인 정보 생성 금지
- SKILL.md에 `## 하지 않는 것` 섹션으로 명시

### 8.2 데이터 확보 한계

- LG전자 글로벌 헤드 카피라이터 책임의 LG 재직 중 카피 원문은 공개 범위가 제한적
- → "지식 구축" 메뉴를 통한 점진적 데이터 축적으로 해결
- → 공개된 캠페인(Life's Good 등)의 카피는 수집 가능

### 8.3 과적합 vs 일반화

- 코퍼스가 적으면 특정 표현만 반복 → Few-shot 다양성 확보 필요
- 코퍼스가 늘어도 LLM 컨텍스트 윈도우 제약 → top-k 검색 + 카테고리 필터로 관련성 높은 것만 선별

### 8.4 연구 기반 한계 인식

`페르소나 지향 텍스트 생성 연구 조사.md`에서 정리된 핵심 발견:
- **표면적 문체 특징은 프롬프팅으로 포착 가능하지만, 깊은 문체 지문(deep stylometric signatures)은 LLM 고유의 것으로 남는 경향** (Mikros 2025)
- → Phase 1~2의 프롬프트 + RAG 접근으로 **실용적 수준**의 문체 재현을 먼저 달성
- → Phase 4에서 LoRA 미세 조정으로 깊은 층위 도전
- **학습 기반 성격 정렬이 프롬프팅보다 일관된 성격 유도에 효과적** (BIG5-CHAT, ACL 2025)
- → 장기적으로 PsychAdapter 방식 검토 근거
