## 1. frontend-v2 UI/UX 개선

### 1.1 Editor 레이아웃 변경

- **2-pane 구조 변경**: 기존 좌(브리프/채팅 탭 전환) ↔ 우(스테이지 결과) → **좌(폼+모든 단계 결과, 80%) ↔ 우(AI 어시스턴트 채팅, 20%)**로 통합
- 좌측 패널에 브리프 입력 → 분석 → 전략 메시지 → 카피 생성 → 리뷰 결과가 세로 스크롤로 연속 표시
- 우측 패널에 AI 어시스턴트 상시 표시
- Case Timeline 제거 (사용자 요청)

### 1.2 사이드바 개선

- 메뉴 버튼을 위쪽 정렬 (기존: flex 채우기 → flex-direction: column + 자연 높이)
- 설정(GearMenu)은 사이드바 최하단 고정 (flex: 1 spacer)
- 글자 크기 및 패딩 원래 값 유지

### 1.3 단계별 버튼 통일

모든 단계의 진행 버튼을 동일한 스타일로 통일 (우측 정렬 + border-top 구분선):

```
① 분석 시작        — 브리핑 하단 (미리보기 버튼도 같은 행에 배치)
② 전략 메시지 생성  — 분석 리포트 하단
③ 카피 후보 생성    — 전략 메시지 하단
④ 선택 카피로 리뷰  — Copy Results 하단
⑤ 리뷰 시작        — 리뷰 조건 설정 하단
⑥ 저장 및 종료      — 리뷰 결과 하단
```

- 분석/전략 메시지 단계: 승인 후에도 "이전 단계"/"전략 재추출" 버튼 유지 (진행 버튼만 숨김)
- 기존 노란 배경 HITL 카드 → 간결한 우측 정렬 버튼으로 교체

### 1.4 자동 스크롤

각 단계 버튼 클릭 시 새 결과물 위치로 왼쪽 패널이 부드럽게 스크롤 (`scrollIntoView`, `behavior: 'smooth'`)

### 1.5 워크플로우 단계 클릭 이동

상단 StepIndicator 클릭 시 해당 단계 섹션으로 스크롤 (active/done 상태만 클릭 가능)

### 1.6 전략 메시지 자동 추출

## 6. 백엔드 버그 수정

### 6.1 SSE 분석 호출 수정

- `/api/v1/campaigns/analyze`가 SSE(StreamingResponse) 반환 → v2가 JSON POST로 호출하던 문제
- `useAnalyze` (JSON) → `useAnalyzeSSE` (readSSE 기반)으로 교체
- 분석 진행 메시지 실시간 표시 (JetBrains Mono)

### 6.2 SSE 리뷰 호출 수정

- `/api/v1/campaigns/review`도 SSE → `useRunReview` → `useRunReviewSSE`로 교체
- 스킬별 결과 실시간 표시

### 6.3 DB 커넥션 풀 안정화

- `pool_pre_ping=True`, `pool_recycle=300` 추가 (stale connection 방지)
- 리뷰 SSE generator에서 `async with async_session() as db:` 자체 세션 사용
- `async_session` import 누락 수정

### 6.4 리뷰 실시간 스트리밍

- `asyncio.gather` (전체 대기) → `asyncio.Queue` 기반 실시간 스트리밍으로 교체
- 각 스킬 완료 즉시 SSE 이벤트 전송 (Vite 프록시 타임아웃 방지)
- Vite 프록시 `/api`, `/events`에 `timeout: 600000` (10분) 추가

### 6.5 LLM 응답 객체 렌더링 에러

- `toneDirection` 등 LLM 필드가 string 대신 object로 반환되는 경우 처리
- `safeText()` 헬퍼: string/number/array/object 어떤 형태든 안전하게 텍스트로 변환
- `analysis-report.tsx`, `strategic-message-view.tsx` 전체 동적 값에 적용

### 6.6 asyncpg 다중 명령 에러

- `CREATE TABLE` 두 개를 하나의 prepared statement에 넣어 발생
- 각각 별도 `conn.execute()` 호출로 분리

## 7. LDAP 로그인 구현

### 7.1 백엔드

- `ldap3`, `redis` 패키지 설치
- `backend/app/auth/` 모듈 신규 생성:
  - `redis_store.py` — Redis 세션 CRUD (URN 키 규칙)
  - `ldap_service.py` — LDAP bind 인증 + displayName/department/email 조회
  - `middleware.py` — AuthContextMiddleware (Cookie → Redis → request.state.auth)
  - `routes.py` — POST /auth/login, POST /auth/logout, GET /auth/me
  - `audit.py` — PostgreSQL 감사 로그 (login_events + session_events)
- `main.py` — 미들웨어 등록, auth 라우터 추가, Redis lifecycle
- `database.py` — auth_login_events, auth_session_events DDL 추가

### 7.2 프론트엔드

- `auth-store.ts` — Zustand (user/setUser/clearUser)
- `login-page.tsx` — ID/PW 입력 폼 (LG Red 테마)
- `protected-layout.tsx` — GET /auth/me guard
- `router.tsx` — /login (public) + ProtectedLayout 감싸기
- `client.ts` — `credentials: 'include'` 추가 (쿠키 전송) + 401 자동 리디렉트
- `app-shell.tsx` — TopbarUser (이름 + 조직 뱃지 + 로그아웃)
- `vite.config.ts` — /auth 프록시 추가

### 7.3 버그 수정
