# 사용자 권한 설정

LG 카피라이팅 에이전트의 인증·권한 구조와 테스트 계정 정리 문서.

## 1. 권한 모델

### 역할(Role)
| 역할 | 설명 | 부여 방식 |
|------|------|-----------|
| `admin` | 관리자 설정/사용 통계/지식 구축 등 운영 기능 접근 | `admin_users` 테이블에 등록된 사용자에 한해 로그인 시점에 세션 `roles`에 주입 |
| `user` (암묵적) | 일반 인증 사용자. 본인 캠페인만 조회/수정/삭제 가능 | LDAP 로그인 성공 시 기본 |

역할은 세션에 스냅샷되며, 관리자 추가/삭제 이후 **다음 로그인**부터 반영된다.

### 핵심 위치
- 모델: `backend/app/models.py` — `AdminUser` 테이블
- 관리자 부트스트랩: `backend/app/main.py:55-70`
- LDAP 인증: `backend/app/auth/ldap_service.py`
- 세션/권한 미들웨어: `backend/app/auth/middleware.py`
- 관리자 엔드포인트: `backend/app/auth/admin_routes.py`

## 2. 권한 강제(Enforcement)

### 의존성 함수
- `require_auth()` — 세션이 없으면 `401`
- `require_admin()` — `ctx.roles`에 `"admin"`이 없으면 `403` (`backend/app/auth/admin_routes.py:62-69`)

### 세션 저장
- Redis 키: `urn:copywriting-agent:{env}:session:{session_id}`
- 쿠키: `SESSION_ID` (httponly, SameSite=Lax — MVP 단계에서는 `secure` 미설정)
- TTL: `AUTH_SESSION_ABSOLUTE_TTL_HOURS` (기본 12시간)

### 공개 경로(인증 제외)
`/`, `/health`, `/docs`, `/openapi.json`, `/auth/login`, `/auth/logout`

## 3. 관리자 전용 엔드포인트

### 관리자 계정 관리 (`/admin/users`)
| Method | Path | 설명 |
|--------|------|------|
| GET | `/admin/users` | 관리자 목록 조회 |
| POST | `/admin/users` | LDAP 사용자 검색 후 관리자 추가 |
| DELETE | `/admin/users/{user_id}` | 관리자 제거 (마지막 1인 제거 방지) |
| POST | `/admin/users/search-ldap` | LDAP 사용자 검색(호출자 비밀번호 필요) |

### 사용 통계 (`/admin/stats`)
| Method | Path | 설명 |
|--------|------|------|
| GET | `/admin/stats/summary` | DAU/MAU, 로그인 성공률 요약 |
| GET | `/admin/stats/dau` | 일간 활성 사용자 추이 (`?days=`) |
| GET | `/admin/stats/mau` | 월간 활성 사용자 추이 (`?months=`) |
| GET | `/admin/stats/users` | 사용자별 사용량 |
| GET | `/admin/stats/departments` | 부서별 사용량 |

### 지식 구축 (`/admin/knowledge`)
지식 베이스 업로드/관리 — 관리자 전용.

### 캠페인 소유권 체크
일반 사용자는 본인이 생성한 캠페인만 접근 가능. 관리자는 모든 캠페인 접근.
- `GET/PUT/DELETE /api/v1/campaigns/{campaign_id}` — 생성자 ≠ 호출자이고 관리자 아니면 `403`
- 대시보드 목록도 관리자가 아니면 본인 캠페인만 노출
- 위치: `backend/app/main.py:1267-1332`

## 5. LDAP 연동

| 항목 | 값 |
|------|------|
| 서버 | `ldaps://lgesaads03.lge.net` |
| Base DN | `dc=lge,dc=net` |
| UPN 도메인 | `lge.com` |
| 바인딩 포맷 | `{username}@lge.com` |
| TLS | MVP: 자체 서명 인증서 허용 (`CERT_NONE`). 프로덕션에서는 CA 번들로 전환 필요 |

로그인 흐름:
1. 사용자 ID(메일 local-part) + 비밀번호 수신
2. LDAP bind로 인증
3. `displayName`, `department`, `mail` 속성 조회
4. `admin_users` 테이블 조회 → 있으면 `roles=["admin"]`
5. Redis에 세션 생성, `SESSION_ID` 쿠키 발급

## 6. 테스트 계정

### 6.1 개발용 로컬 계정 (LDAP 우회)
`ENABLE_DEV_ACCOUNTS=true`일 때만 동작. 실제 LDAP를 타지 않는다.

| 항목 | 값 |
|------|------|
| Username | `user` |
| Password | `1234` |
| Display Name | 테스트 사용자 |
| Department | 테스트팀 |
| Email | `user@lge.com` |
| 권한 | 일반 사용자 (admin_users에 별도 등록하지 않는 한 `user`) |

- 정의 위치: `backend/app/auth/ldap_service.py:62-69`
- 분기 위치: `backend/app/auth/ldap_service.py:89-98`
- **운영 배포 시 반드시 `ENABLE_DEV_ACCOUNTS=false`로 전환할 것.**

### 6.2 초기 관리자 부트스트랩
`admin_users` 테이블이 비어있을 때 서버 기동 시 1회만 삽입.

```
INITIAL_ADMIN_USER_IDS=jounghyun.kim
```

- 현재 값: `jounghyun.kim` (`backend/.env:16`)
- 콤마로 복수 지정 가능
- 테이블에 이미 레코드가 있으면 동작하지 않음 (중복 삽입 방지)
- 위치: `backend/app/main.py:56-70`

### 6.3 실제 LDAP 계정
임의의 LG 임직원 계정(예: `jounghyun.kim` + LG 비밀번호)으로 로그인 가능. `INITIAL_ADMIN_USER_IDS`에 포함되거나 관리자 페이지에서 추가된 계정은 admin 권한을 받는다.

## 7. 감사 로그

서버 기동 시 두 테이블이 생성된다.

| 테이블 | 용도 |
|--------|------|
| `auth_login_events` | 로그인 시도 (성공/실패, 사용자명, source_ip, department 스냅샷) |
| `auth_session_events` | 세션 이벤트 (created / accessed / logout) |

DAU 집계는 미들웨어에서 Redis dedup 후 DB에 fire-and-forget 기록 (`backend/app/auth/middleware.py:99-156`).

## 9. 운영 체크리스트

- [ ] 배포 전 `ENABLE_DEV_ACCOUNTS=false`

- [ ] `admin_users`에서 마지막 1인 제거 방지 로직 확인
