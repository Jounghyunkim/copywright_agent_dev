# Admin Management Skill 적용 문서

## 개요

Copywriting Agent에 관리자 역할(role) 관리 시스템을 구현했습니다.
LDAP 실시간 검색 기반으로 관리자 추가/제거하고, 역할 기반 엔드포인트 보호 및 조건부 UI를 제공합니다.

---

## 확정 변수

| 변수 | 값 |
|------|-----|
| `ADMIN_ROLE_NAME` | `admin` |
| `INITIAL_ADMIN_USER_IDS` | `jounghyun.kim` |
| `USER_ID_TYPE` | mail local-part (예: `jounghyun.kim`) |
| 사용자 검색 방식 | **LDAP 실시간 검색** (관리자 본인 자격증명 사용, 서비스 계정 불필요) |

---

## 아키텍처

```
[Bootstrap]   .env INITIAL_ADMIN_USER_IDS → admin_users 테이블 시딩 (최초 1회, 멱등)
                                                │
[Login]       LDAP 인증 성공 → admin_users 조회 → roles에 "admin" 추가 + 프로필 갱신
                                                │
[Session]     Redis 세션에 roles 저장 (TTL 30분/12시간)
                                                │
[Admin UI]    관리자 비밀번호 입력 → LDAP 검색 → 관리자 추가/제거
                                                │
[Protection]  require_admin dependency → admin 역할 없으면 403
```

### 핵심 설계 결정

1. **서비스 계정 불필요** — 관리자 본인 자격증명으로 LDAP 검색 (보안성 높음, 감사 추적 가능)
2. **역할 반영은 로그인 시** — 관리자 추가/제거 후 다음 로그인부터 반영
3. **프로필 자동 갱신** — 관리자 로그인 시마다 LDAP에서 이름/조직/이메일 최신화 (부트스트랩 시 ID로 저장되는 문제 해결)
4. **마지막 관리자 보호** — admin_count <= 1이면 제거 불가 (409 Conflict)
5. **부트스트랩 멱등성** — 테이블이 비어있을 때만 시딩

## 데이터베이스 스키마

```sql
CREATE TABLE IF NOT EXISTS admin_users (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id VARCHAR(128) NOT NULL UNIQUE,      -- 로그인 ID와 동일 형식 (mail local-part)
  display_name VARCHAR(255) NOT NULL,        -- LDAP displayName
  department VARCHAR(255) NULL,              -- LDAP department
  email TEXT NULL,                           -- LDAP mail
  added_by VARCHAR(128) NOT NULL,            -- 추가한 사람의 user_id (감사 추적)
  created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE UNIQUE INDEX IF NOT EXISTS idx_admin_users_user_id
  ON admin_users (user_id);
```

- SQLAlchemy ORM으로 `Base.metadata.create_all`에서 자동 생성
- `added_by = 'system_bootstrap'`은 부트스트랩으로 추가된 초기 관리자

```http
POST /admin/users
Content-Type: application/json

{
  "user_id": "gildong.hong",
```

- `404 admin_not_found`

## 로그인 플로우 통합

`backend/app/auth/routes.py`:

```python
@router.post("/login", response_model=LoginResponse)
async def login(body: LoginRequest, request: Request, response: Response):
    # 1. LDAP 인증
    user_info = authenticate(body.username, body.password)

    # 2. Admin 역할 확인 + 프로필 갱신
    roles: list[str] = []
    async with async_session() as db:
        result = await db.execute(
            select(AdminUser).where(AdminUser.user_id == user_info.user_id)
        )
        admin_record = result.scalar_one_or_none()
        if admin_record:
            roles.append("admin")
            # 매 로그인 시 LDAP 데이터로 프로필 갱신
            admin_record.display_name = user_info.display_name
            admin_record.department = user_info.department
            admin_record.email = user_info.email
            await db.commit()

    # 3. Redis 세션 생성 (roles 포함)
    session_id = await create_session(
        user_id=user_info.user_id,
        display_name=user_info.display_name,
        department=user_info.department,
        email=user_info.email,
        roles=roles,  # ← ["admin"] 또는 []
    )

    # 4. Cookie 설정 (기존과 동일)
    response.set_cookie(key=AUTH_COOKIE_NAME, value=session_id, ...)
    return LoginResponse(...)
```

## 부트스트랩 시딩

`backend/app/main.py` lifespan:

```python
@asynccontextmanager
async def lifespan(app: FastAPI):
    await init_db()
    # ... Redis 연결 확인 ...

    # 초기 관리자 부트스트랩 (테이블 비어있을 때만)
    initial_admins = os.getenv("INITIAL_ADMIN_USER_IDS", "")
    if initial_admins:
        async with async_session() as db:
            count = (await db.execute(func.count(AdminUser.id))).scalar() or 0
            if count == 0:
                for uid in initial_admins.split(","):
                    uid = uid.strip()
                    if uid:
                        db.add(AdminUser(
                            user_id=uid,
                            display_name=uid,  # placeholder → 첫 로그인 시 LDAP로 갱신
                            added_by="system_bootstrap",
                        ))
                await db.commit()
                print(f"Bootstrapped initial admins: {initial_admins}")
    yield
    await close_redis()
```

**특징**:
- 멱등적(idempotent) — 이미 관리자가 있으면 건너뜀
- 이름은 ID로 초기화 → 첫 로그인 시 LDAP로 실제 이름/조직/이메일 갱신

## require_admin 의존성

`backend/app/auth/admin_routes.py`:

```python
from .middleware import AuthContext, require_auth

def require_admin(ctx: AuthContext = Depends(require_auth)) -> AuthContext:
    """Require authenticated user with 'admin' role."""
    if "admin" not in ctx.roles:
        raise HTTPException(status_code=403, detail="admin_required")
    return ctx

# 사용 예시
@router.get("/admin/users")
async def list_admins(ctx: AuthContext = Depends(require_admin)):
    ...
```

## 프론트엔드 페이지 구조

`/admin/users` 페이지 (`pages/admin/admin-users-page.tsx`)

### Layout

```
┌─ 관리자 설정 ───────────────────────────────┐
│ 관리자 추가/제거 및 사용자 검색              │
├─ 현재 관리자 (N명) ─────────────────────────┤
│ 이름 | ID | 조직 | 추가일 | 추가자 | [제거] │
│ ...                                          │
├─ 사용자 검색 (LDAP) ────────────────────────┤
│ [비밀번호 게이트 OR 검색 모드]              │
└──────────────────────────────────────────────┘
```

### Phase 1: 비밀번호 게이트

- LDAP 검색을 위해 본인 비밀번호 입력
- Enter 또는 "확인" 클릭 → `passwordVerified = true`

### Phase 2: 검색 모드

- 검색 입력 (2글자 이상, 500ms debounce)
- 결과 테이블: 이름 | ID | 조직 | [추가 버튼 / 관리자 뱃지]
- "비밀번호 재입력" 링크로 Phase 1로 복귀

### 상태 관리

```typescript
const [admins, setAdmins] = useState<AdminUser[]>([])
const [password, setPassword] = useState('')
const [passwordVerified, setPasswordVerified] = useState(false)
const [query, setQuery] = useState('')
const [results, setResults] = useState<LdapResult[]>([])
const [searching, setSearching] = useState(false)
const [searchError, setSearchError] = useState<string | null>(null)
const [actionLoading, setActionLoading] = useState<string | null>(null)
```

### Debounce 검색

```typescript
useEffect(() => {
  if (debounceRef.current) clearTimeout(debounceRef.current)
  if (!passwordVerified || query.trim().length < 2) {
    setResults([])
    return
  }
  debounceRef.current = setTimeout(() => doSearch(query.trim()), 500)
  return () => clearTimeout(debounceRef.current)
}, [query, passwordVerified])
```

### 에러 처리

| API 에러 | 프론트 동작 |
|----------|-----------|
| 401 (invalid_ldap_password) | passwordVerified=false, 비밀번호 재입력 유도 |
| 503 (ldap_unavailable) | "LDAP 서버에 연결할 수 없습니다." |
| 409 (user_already_admin) | alert: "이미 관리자로 등록된 사용자입니다." |
| 409 (cannot_remove_last_admin) | alert: "마지막 관리자는 제거할 수 없습니다." |
| 404 (admin_not_found) | alert: "관리자를 찾을 수 없습니다." |

### 제거 시 확인 다이얼로그

```typescript
const isSelf = admin.user_id === currentUser?.user_id
const msg = isSelf
  ? `자신(${displayName})을 관리자에서 제거하시겠습니까?\n다음 로그인부터 관리자 권한이 해제됩니다.`
  : `${displayName}(${userId})을 관리자에서 제거하시겠습니까?`
```

## 알려진 주의사항 (스킬 Pitfalls 대응)

| 주의사항 | 우리 구현에서의 대응 |
|---------|-------------------|
| user_id 형식 불일치 | 로그인 ID와 admin_users.user_id 모두 mail local-part (예: `jounghyun.kim`) |
| 부트스트랩 이름이 ID로 표시 | 매 로그인 시 LDAP 프로필로 갱신 |
| LDAP 검색 타임아웃 | `receive_timeout = max(connect_timeout * 3, 15)`, `time_limit=15` |
| LDAP 검색 범위 | `(objectCategory=person)(objectClass=user)` 필터 적용 |
| 서비스 계정 관리 부담 | 관리자 본인 자격증명 사용 |
| 역할 반영 지연 | 다음 로그인 시 반영 (UI에서 "다음 로그인부터 반영됩니다" 안내) |
| 마지막 관리자 삭제 | `admin_count <= 1`이면 409 반환 |

## MVP vs Deferred

### MVP (구현 완료)

- ✅ `admin_users` 테이블 + ORM 모델
- ✅ 부트스트랩 시딩 (`INITIAL_ADMIN_USER_IDS`, 멱등적)
- ✅ 로그인 시 admin 역할 부여 + 프로필 자동 갱신
- ✅ `require_admin` FastAPI dependency
- ✅ LDAP 사용자 검색 (관리자 본인 자격증명)
- ✅ CRUD 4개 API (list, search-ldap, add, remove)
- ✅ 마지막 관리자 보호
- ✅ 관리자 설정 페이지 (PW 게이트 + 검색 + 추가/제거)
- ✅ admin 조건부 GearMenu

### Deferred (향후 구현)

- 역할 다중화 (viewer, reviewer 등)
- Redis 세션 실시간 갱신 (관리자 추가/제거 즉시 반영)
- 관리자 활동 감사 로그 테이블 (`admin_action_log`)
- 관리자 인증 로그 (언제 누가 로그인했는지)
- 기존 엔드포인트에 `require_admin` 선택적 적용 (스킬 CRUD, 캠페인 삭제 등)
- UI: 사용자 목록 페이지 (전체 가입자 관리)
